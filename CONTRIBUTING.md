# Contributing to Device Assessment and Vulnerability Exposure (DAVE)

Thanks for your interest in this project.

## Project Description

The **Device Assessment and Vulnerability Exposure (DAVE)** is a comprehensive platform designed to manage medical device cybersecurity, FDA compliance, and vulnerability tracking for healthcare organizations. DAVE provides asset monitoring, vulnerability assessment, and risk assessment capabilities specifically tailored for the healthcare industry.

### Key Features

- **Medical Device Security Management** - Comprehensive asset tracking and vulnerability management
- **FDA Compliance Monitoring** - Automated recall monitoring and device mapping
- **Vulnerability Assessment** - SBOM processing and CVE tracking
- **Risk Prioritization** - Risk scoring and remediation planning
- **Compliance Reporting** - Automated compliance reports and dashboards
- **Enterprise Security** - Role-based access control and audit logging

### Technology Stack

- **Frontend**: PHP 8.0+, HTML5, CSS3, JavaScript (ES6+)
- **Backend**: Python 3.9+, PHP 8.0+
- **Database**: PostgreSQL 14+
- **Web Server**: Apache 2.4+ or Nginx 1.18+
- **OS**: Linux (Ubuntu 22.04 LTS or later recommended)

3. **Create a feature branch** for your contribution

5. **Submit a pull request** with a clear description of your changes

## Development Process

### Coding Standards

* **PHP**: Follow PSR-12 coding standards
* **Python**: Follow PEP 8 style guide
* **JavaScript**: Use ES6+ features and consistent formatting
* **SQL**: Use lowercase keywords and proper indentation

### Testing

* Test your changes thoroughly before submitting
* Ensure existing functionality is not broken
* Include unit tests for new features where applicable

### Commit Messages

* Use clear, descriptive commit messages
* Reference issue numbers when applicable
* Follow the format: `[Component] Brief description`

Example:
```
[API] Add endpoint for device vulnerability export
[UI] Fix pagination on recalls page
[DB] Add migration for new risk_priority table
```

### Pull Request Process

1. Update documentation for any changed functionality
2. Ensure your code follows the project's coding standards
3. Test your changes in a development environment
4. Provide a clear description of the changes in your PR
5. Link to any relevant issues
